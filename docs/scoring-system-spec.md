# 综合评分与排除岗位规范

## 目标

为不同用户提供一致的评分框架，同时允许每位用户独立声明绝不投递的职业类型。源码不得预设个人目标方向或排除岗位。

## 综合评分

AI 参考用户配置、所有已勾选简历、完整 JD、岗位门槛、地点和机会质量，给出最终 0-100 分：

| 维度 | 分值 |
| --- | ---: |
| 方向相关性 | 30 |
| 简历证据 | 25 |
| 岗位门槛 | 20 |
| 地理位置 | 10 |
| 机会质量 | 15 |

扩展只把分数限制在 0-100，并与用户设置的最低分比较。与目标方向、简历主线和可迁移能力都基本无关的岗位，即使门槛低，也应低于 50 分。

## 配置边界

- `目标方向/关键词`：用户希望寻找的岗位方向，是评分强信号。
- `绝不投递岗位/职业类型`：明确命中后无条件跳过，优先级高于分数。
- `额外分析提示词`：补充初级友好度、工作方式、行业偏好等个性化条件。
- 所有字段默认留空或使用中性默认值，只保存在当前用户浏览器中。

## 排除岗位语义规则

1. 先依据标题和完整 JD 概括岗位主要职业类型。
2. 将主要职业类型与用户排除项逐项进行完整语义比较。
3. 只有岗位核心工作明确属于排除职业类型时才算命中。
4. 共享宽泛词不算命中：
   - 技术支持不等于客户支持。
   - 产品运营不等于直播运营。
   - 市场策划不等于电话销售。
   - 客户服务不等于保险销售。
5. 少量附带任务、公司行业相似或标题含糊时不得误判为排除。
6. 排除列表为空时永远不触发排除。

## 输出契约

```json
{
  "score": 0,
  "decision": "recommend|manual_review|skip",
  "excluded": false,
  "exclusion_match": "",
  "exclusion_reason": "",
  "occupation_family": "",
  "target_alignment": "direct|transferable|unrelated|unclear",
  "reasons": [],
  "risks": [],
  "resume_tips": [],
  "location_fit": "good|acceptable|unclear|poor",
  "greeting": "",
  "qa": []
}
```

扩展处理规则：

1. `excluded=true`：强制 `decision=skip`，最终分最高为 19。
2. `excluded=false` 且分数达到用户最低分：允许进入自动沟通候选。
3. 未达到最低分：不自动沟通。

## 测试要求

- 排除配置能保存、读取并进入 AI 请求。
- 默认排除列表为空，不影响普通岗位。
- 明确命中的排除岗位即使模型返回高分也必须跳过。
- 相邻但不同的职业类型不得因为共享宽泛词被排除。
- 未排除岗位只按最终分和用户最低分判断。
- 已勾选简历为空时不得回退到未勾选简历。
